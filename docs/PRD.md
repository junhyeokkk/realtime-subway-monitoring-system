# PRD
# MetroWatch
## 실시간 도시철도 운영·혼잡 관제 시스템

---

# 1. 프로젝트 개요

## 프로젝트명
MetroWatch

---

## 프로젝트 설명

MetroWatch는 공공데이터 기반 실시간 도시철도 운영·혼잡 관제 시스템이다.

도시철도 운행 정보, 역사별 승하차 데이터, 혼잡도 데이터를 수집·분석하여:

- 시민에게는 실시간 열차 정보 및 혼잡도를 제공하고
- 운영자에게는 혼잡 구간 및 이상 상황을 관제할 수 있는 기능을 제공한다.

본 프로젝트는 단순 정보 조회 서비스가 아니라:

- 실시간 데이터 수집
- 데이터 검증
- 장애 감지
- 운영 로그 관리
- 관리자 관제

기능을 포함하는 운영형 시스템 구축을 목표로 한다.

---

# 2. 프로젝트 배경

현재 도시철도 관련 공공데이터는 다양한 형태로 제공되고 있으나:

- 실시간 운영 관점에서 통합적으로 활용하기 어렵고
- 시민은 혼잡도 및 운영 상태를 직관적으로 확인하기 어렵다.

또한 운영 측면에서도:
- 혼잡 구간 분석
- 데이터 수집 상태 관리
- API 장애 감지
- 이상 데이터 탐지

기능을 통합적으로 관리하기 어렵다.

MetroWatch는 공공데이터를 기반으로 도시철도 운영 상태를 실시간으로 수집·분석하고 시각화하여 시민 편의 및 운영 효율 향상을 목표로 한다.

# 5. 핵심 기능

# 5-1. 실시간 열차 도착 정보

## 설명
공공 API 기반 실시간 열차 도착 정보를 제공한다.

## 주요 기능
- 역사 검색
- 상·하행 열차 정보 제공
- 도착 예정 시간 제공
- Redis 기반 캐싱 처리

---

# 5-2. 역사별 혼잡도 분석

## 설명
승하차 데이터를 기반으로 역사별 혼잡도를 계산한다.

## 주요 기능
- 시간대별 혼잡도 조회
- 혼잡 단계 계산
- 혼잡 TOP 역사 조회

## 혼잡 단계 예시

| 단계 | 설명 |
|---|---|
| LOW | 여유 |
| NORMAL | 보통 |
| HIGH | 혼잡 |
| VERY_HIGH | 매우 혼잡 |

---

# 5-3. 운영 관제 대시보드

## 설명
운영자가 전체 운영 상태를 확인할 수 있는 관리자 기능 제공.

## 주요 기능
- 데이터 수집 상태 확인
- API 장애 상태 모니터링
- 혼잡 현황 시각화
- 운영 로그 조회

---

# 5-4. 장애 감지 및 알림

# 10. 주요 설계 포인트

# Collector 계층 분리

공공 API 수집 계층을 별도로 분리하여:

- 장애 재시도
- 데이터 검증
- 수집 로그 관리

가 가능하도록 설계하였다.

---

실시간 데이터는 Redis TTL 캐시를 활용하여 조회 성능을 최적화하였다.

---

# 운영형 시스템 설계

단순 CRUD 서비스가 아니라:

- 운영 로그
- 장애 추적
- 관리자 관제
- 데이터 정합성

관점을 고려하여 설계하였다.
